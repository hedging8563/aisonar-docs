## 推荐流程

1. 调用 [模型列表](/zh/api-reference/models/list-models)，带上 `recommended_for=music`。
2. 用 [创建音乐](/zh/api-reference/music/create-music) 创建任务，传 `prompt`，并按需传 `title`、`tags`、`action`、`mv` 或续写设置。
3. 保存返回的 `id` / `task_id` 和 `poll_url`。
4. 轮询到 `completed` 或 `failed`，再读取 `audio_url`、可选 `video_url`、`title` 和 `lyrics`。

## 模型发现

音乐生成是异步任务。不要依赖上游供应商名称或上游任务 ID；AI Sonar 会返回标准化的任务 ID 和轮询 URL。

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## 创建音乐任务

`action: "MUSIC"` 用于歌曲或纯音乐，具体取决于提示词和模型。`action: "LYRICS"` 只用于所选模型支持的歌词生成流程。`continue_clip_id` 和 `continue_at` 只用于模型契约明确支持的续写场景。

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## 轮询结果

始终优先使用返回的 `poll_url`。稳定兜底入口是 [查询任务状态](/zh/api-reference/tasks/get-task-status)。完成后的音乐任务可能包含 `audio_url`、`video_url`、`title`、`lyrics` 和模型归一化后的元数据。

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
