<Note>
Dieser Leitfaden richtet sich an Nutzer von **selbst gehostetem OpenClaw**, die TokenLab als ihren KI-Provider verbinden möchten.
</Note>

## Übersicht

Für aktuelle OpenClaw-Versionen wird empfohlen, TokenLab über `models.providers` zu konfigurieren.

Wenn Sie schnell loslegen möchten, reicht es aus, nur `tokenlab` zu konfigurieren. Fügen Sie die anderen Provider nur hinzu, wenn Sie explizit **Responses API**, **Claude native**, **Gemini native** oder **MiniMax native** Verhalten benötigen.

| Anbieter | OpenClaw `api` | Am besten für | baseUrl |
|----------|----------------|---------------|---------|
| `tokenlab` | `openai-completions` | GPT, DeepSeek, Qwen und die meisten OpenAI-kompatiblen Aufrufe | `https://api.tokenlab.sh/v1` |
| `tokenlab-responses` | `openai-responses` | OpenAI Responses-Workflows, die `/v1/responses`-Semantik erwarten | `https://api.tokenlab.sh/v1` |
| `tokenlab-claude` | `anthropic-messages` | Natives Claude Messages API | `https://api.tokenlab.sh` |
| `tokenlab-gemini` | `google-generative-ai` | Native Gemini-API-Format | `https://api.tokenlab.sh` |
| `tokenlab-minimax` | `anthropic-messages` | Native MiniMax-Routing | `https://api.tokenlab.sh` |

<Warning>
Verwenden Sie den Suffix `/v1` nur für `openai-completions` und `openai-responses`.

Native Provider wie `anthropic-messages` und `google-generative-ai` sollten `https://api.tokenlab.sh` ohne `/v1` verwenden, andernfalls kann OpenClaw den falschen Upstream-Pfad konstruieren.
</Warning>

## Voraussetzungen

- Eine selbst gehostete OpenClaw-Instanz
- Einen TokenLab API-Schlüssel — [Hier erhalten](https://tokenlab.sh/dashboard)

## Konfiguration

Bearbeiten Sie Ihre OpenClaw-Konfiguration:

- **Selbst gehostet**: `~/.openclaw/openclaw.json`

Fügen Sie TokenLab-Provider unter `models.providers` hinzu:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "tokenlab-claude/claude-sonnet-4-6"
      }
    }
  },
  models: {
    mode: "merge",
    providers: {
      tokenlab: {
        api: "openai-completions",
        baseUrl: "https://api.tokenlab.sh/v1",
        apiKey: "sk-your-tokenlab-key",
        models: [
          { id: "gpt-4o", name: "GPT-4o" },
          { id: "deepseek-r1", name: "DeepSeek R1" },
          { id: "qwen3-32b", name: "Qwen 3 32B" }
        ]
      },
      "tokenlab-responses": {
        api: "openai-responses",
        baseUrl: "https://api.tokenlab.sh/v1",
        apiKey: "sk-your-tokenlab-key",
        models: [
          { id: "gpt-4o", name: "GPT-4o (Responses)" },
          { id: "gpt-5.2", name: "GPT-5.2 (Responses)" }
        ]
      },
      "tokenlab-claude": {
        api: "anthropic-messages",
        baseUrl: "https://api.tokenlab.sh",
        apiKey: "sk-your-tokenlab-key",
        models: [
          { id: "claude-sonnet-4-6", name: "Claude Sonnet 4.6" },
          { id: "claude-opus-4-6", name: "Claude Opus 4.6" }
        ]
      },
      "tokenlab-gemini": {
        api: "google-generative-ai",
        baseUrl: "https://api.tokenlab.sh",
        apiKey: "sk-your-tokenlab-key",
        models: [
          { id: "gemini-2.5-flash", name: "Gemini 2.5 Flash" },
          { id: "gemini-3-flash-preview", name: "Gemini 3 Flash Preview" }
        ]
      },
      "tokenlab-minimax": {
        api: "anthropic-messages",
        baseUrl: "https://api.tokenlab.sh",
        apiKey: "sk-your-tokenlab-key",
        models: [
          { id: "minimax-m1", name: "MiniMax M1" }
        ]
      }
    }
  }
}
```

<Info>
Alle 5 Provider verwenden denselben **API-Schlüssel**. Sie benötigen nur ein TokenLab-Konto.
</Info>

<Info>
Die oben gezeigten `models`-Arrays enthalten nur gängige Beispiele. Fügen Sie je nach Bedarf weitere Modell-IDs zu jedem Provider hinzu.
</Info>

## Verwendung von Modellen

OpenClaw referenziert Modelle weiterhin im Format `provider/model`:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "tokenlab-gemini/gemini-2.5-flash"
      }
    }
  }
}
```

### Modellbeispiele

| Anbieter | Modellreferenz | Beschreibung |
|----------|----------------|--------------|
| `tokenlab` | `tokenlab/gpt-4o` | OpenAI-kompatibler Pfad |
| `tokenlab` | `tokenlab/deepseek-r1` | DeepSeek-Reasoning-Modell |
| `tokenlab-responses` | `tokenlab-responses/gpt-4o` | Responses API-Pfad |
| `tokenlab-claude` | `tokenlab-claude/claude-sonnet-4-6` | Native Claude Messages-Route |
| `tokenlab-gemini` | `tokenlab-gemini/gemini-2.5-flash` | Native Gemini-Route |
| `tokenlab-minimax` | `tokenlab-minimax/minimax-m1` | Native MiniMax-Route |

Alle verfügbaren Modelle finden Sie unter [tokenlab.sh/models](https://tokenlab.sh/de/models).

## Wann Sie welchen Provider verwenden sollten

- `tokenlab`: Standardwahl für die meisten allgemeingültigen Agent- und Chat-Anwendungsfälle.
- `tokenlab-responses`: Verwenden Sie diesen, wenn Ihr OpenClaw-Workflow explizit auf OpenAI Responses-Semantik angewiesen ist.
- `tokenlab-claude`: Verwenden Sie diesen, wenn Sie Claudes native Messages-Verhalten möchten.
- `tokenlab-gemini`: Verwenden Sie diesen, wenn Sie Gemini-native Anfrage-/Antwort-Formatierung oder bestehende Gemini-Integrationen benötigen.
- `tokenlab-minimax`: Verwenden Sie diesen, wenn Sie MiniMax über dessen native Route nutzen möchten.

Wenn Sie kein Gemini-native Verhalten benötigen, können Sie Gemini-Modelle weiterhin über `tokenlab/gemini-*` über die OpenAI-kompatible Route aufrufen.

## Häufige Fehler

<AccordionGroup>
  <Accordion title="Verwendet noch das alte Top-Level `providers`-Array">
    Die aktuelle OpenClaw-Dokumentation verwendet `models.providers`. Wenn Sie weiterhin das ältere Top-Level-`providers`-Array-Format beibehalten, kann OpenClaw die Konfiguration ignorieren oder die Provider-Präfixe nicht wie erwartet auflösen.
  </Accordion>
  <Accordion title="Vergessen von /v1 bei `tokenlab-responses`">
    `openai-responses` mappt auf TokenLabs `/v1/responses`-Pfad, daher muss `tokenlab-responses` `https://api.tokenlab.sh/v1` verwenden.
  </Accordion>
  <Accordion title="Hinzufügen von /v1 zu `tokenlab-claude`, `tokenlab-gemini` oder `tokenlab-minimax`">
    `anthropic-messages` und `google-generative-ai` sollten `https://api.tokenlab.sh` ohne `/v1` verwenden. Das Hinzufügen von `/v1` kann zu falschen Anforderungs-Pfaden führen.
  </Accordion>
  <Accordion title="Unterstützt OpenClaw weiterhin native Gemini?">
    Ja. Die aktuelle OpenClaw-Dokumentation enthält weiterhin den eingebauten `google`-Provider und unterstützt außerdem benutzerdefinierte Provider mit `api: "google-generative-ai"`. `tokenlab-gemini` bleibt daher eine gültige native Gemini-Route für OpenClaw-Nutzer.
  </Accordion>
</AccordionGroup>

## Einrichtung verifizieren

Nach dem Speichern der Konfiguration starten Sie Ihre OpenClaw-Instanz neu und testen Sie mit einer einfachen Nachricht. Wenn Sie eine Antwort erhalten, ist der Provider korrekt konfiguriert.

```bash
# Selbst gehostet: Dienst neu starten
systemctl --user restart openclaw    # Linux
launchctl stop cc.tokenlab.openclaw && launchctl start cc.tokenlab.openclaw  # macOS
```
