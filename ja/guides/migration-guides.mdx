## 概要

このガイドでは、「移行ガイド」を本番で使う方法を説明します。アプリケーションをプロバイダー実装詳細に結び付けません。

AI Sonar の公開 API、モデルメタデータ、文書化されたレスポンスフィールドを契約として扱います。

## 現在の動作

- リリース前に次の値を確認します: base URL, API key, model ID。
- プロバイダー固有の状態 URL、ルーティング識別子、非公開の実行メタデータには依存しないでください。
- リトライは冪等にし、重複課金される作業を作らないでください。

## 本番チェックリスト

- ログはサポートに役立つ範囲に留め、秘密情報、私有ファイル、prompt、ユーザーメディアは保存しません。
- サポート依頼には、秘匿化したリクエスト形状、エンドポイント、モデル、時刻、公開識別子を含めます。
- 本番トラフィックを移す前にステージングでフローを検証します。
