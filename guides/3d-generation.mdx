## Recommended flow

1. Call [List models](/api-reference/models/list-models) with `recommended_for=3d`.
2. Use [Create 3D](/api-reference/3d/create-3d) with a text `prompt`; add `image` or `image_url` only for image-to-3D models that support it.
3. Choose optional fields such as `format`, `quality`, `style`, or `seed` according to the selected model contract.
4. Poll `poll_url` until the asset is ready, then download `model_url` or format-specific URLs such as `glb_url` / `fbx_url`.

## Model discovery

3D generation is task-based. Treat the model catalog as the source of truth instead of assuming one provider or one fixed model.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Create a 3D task

Text-to-3D requests require a `prompt`. Image-to-3D requests add either `image` base64 data or `image_url`. Use `format` and `quality` only where the model contract exposes them.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Polling results

Use the returned `poll_url` first. The stable fallback is [Get task status](/api-reference/tasks/get-task-status). Completed 3D tasks return `model_url` and may include format-specific fields such as `glb_url` or `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
