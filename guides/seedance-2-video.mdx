title: "Seedance 2.0 Video Models"
description: "Choose between Seedance 2.0, Seedance 2.0 Fast, and Seedance 2.0 Mini for video generation."
---

Seedance 2.0 is a video model family for text-to-video, image-to-video, reference-to-video, video editing, and video extension workflows. Use the model IDs shown in AI Sonar, such as `seedance-2.0`, then choose the workflow with `operation`.

## Model Choice

| Model | Best for | Supported resolutions | Reference inputs |
|-------|----------|--------------------|------------------|
| `seedance-2.0` | Highest quality and 4K output | `480p`, `720p`, `1080p`, `4k` | Up to 9 reference images plus up to 3 reference videos and 3 reference audios for `reference-to-video` |
| `seedance-2.0-fast` | Faster, lower-cost 480p/720p output | `480p`, `720p` | Up to 9 reference images plus up to 3 reference videos and 3 reference audios for `reference-to-video` |
| `seedance-2.0-mini` | Lowest-cost 480p/720p output | `480p`, `720p` | Supports the Seedance 2.0 operation family; read the current model detail before relying on exact reference count limits |

`seedance-2.0-fast` and `seedance-2.0-mini` support only `480p` / `720p`; requests with `1080p` or `4k` return a clear error for those models.

## Supported Operations

The Seedance 2.0 family supports:

| Operation | Typical inputs |
|-----------|----------------|
| `text-to-video` | `prompt` |
| `image-to-video` | `prompt`, `image_url` or compatible image input |
| `start-end-to-video` | `prompt`, `start_image`, `end_image` |
| `reference-to-video` | `prompt`, `reference_images`, optional `video_urls`, optional `audio_urls` |
| `video-to-video` | `prompt`, `video_url` or compatible video input |
| `video-extension` | `task_id` or model-specific continuation input |

Pure audio-only and text-plus-audio-only Seedance requests are not supported. Use audio as a reference input only in supported `reference-to-video` flows.

## Material Groups And Material IDs

Seedance 2.0 material assets are organized in AI Sonar material groups. Use `library_type: "aigc_avatar"` for virtual/avatar materials and `library_type: "liveness_face"` for certified real-person materials. Create or list groups with the [material asset group endpoints](/api-reference/video/create-material-asset-group), then upload assets with [Create Material Asset](/api-reference/video/create-material-asset).

For the full upload request body, status fields, and examples, use the [Create Material Asset](/api-reference/video/create-material-asset) reference.

The `group_id` field is the AI Sonar material group ID returned by these endpoints. If you omit it, AI Sonar uses the current organization's default group for the selected `library_type`. After an uploaded asset becomes `ACTIVE`, pass its returned `id` as `material_asset_id` or include it in `material_asset_ids` when calling `POST /v1/videos/generations`. Material assets count toward Seedance's 9 image-reference limit.

## 4K Reference-To-Video

Use `seedance-2.0` when you need 4K output. The base model supports `4k`, including in `reference-to-video` requests.

```bash
curl -X POST "https://api.aisonar.dev/v1/videos/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0",
    "operation": "reference-to-video",
    "prompt": "Keep the same product identity and lighting while creating a slow cinematic rotation.",
    "reference_images": [
      "https://example.com/product-front.jpg",
      "https://example.com/product-side.jpg"
    ],
    "video_urls": ["https://example.com/motion-reference.mp4"],
    "duration": 6,
    "resolution": "4k",
    "aspect_ratio": "16:9"
  }'
```

## Fast And Mini Requests

Use `seedance-2.0-fast` when latency matters, and `seedance-2.0-mini` when the lowest cost tier is more important than maximum quality.

```bash
curl -X POST "https://api.aisonar.dev/v1/videos/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "seedance-2.0-fast",
    "operation": "text-to-video",
    "prompt": "A clean product reveal on a white studio background, gentle camera push-in.",
    "duration": 5,
    "resolution": "720p",
    "aspect_ratio": "16:9"
  }'
```

## Pricing And Billing

Seedance 2.0 pricing is token-based and depends on output resolution and whether the request includes video input.

Use the [Pricing API](/api-reference/pricing/get-pricing) or [Models API](/api-reference/models/list-models) for the current billable price before hard-coding product UI.

## Async Results

Video generation is asynchronous. Follow the `poll_url` returned by the create request, or use `GET /v1/tasks/{id}` with the returned task ID.

See [Create Video](/api-reference/video/create-video), [Get Task Status](/api-reference/tasks/get-task-status), and [Cancel Task](/api-reference/tasks/cancel-task) for endpoint details.
