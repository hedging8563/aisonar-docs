## Overview

This guide explains how to use Provider Routing & Reliability in production without binding your application to provider implementation details.

Use AI Sonar public APIs, model metadata, and documented response fields as the contract.

## Current behavior

- Review these values before release: model contract, route availability, retry policy.
- Do not depend on provider-specific status URLs, routing identifiers, or private execution metadata.
- Make retries idempotent and avoid creating duplicate billable work.

## Production checklist

- Keep logs useful for support without storing secrets, private files, prompts, or user media.
- Support tickets should include sanitized request shape, endpoint, model, timestamp, and public identifiers.
- Verify the flow in staging before moving production traffic.
