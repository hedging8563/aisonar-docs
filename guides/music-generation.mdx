## Recommended flow

1. Call [List models](/api-reference/models/list-models) with `recommended_for=music`.
2. Use [Create music](/api-reference/music/create-music) to create a task with `prompt` and optional fields such as `title`, `tags`, `action`, `mv`, or continuation settings.
3. Store the returned `id` / `task_id` and `poll_url`.
4. Poll until the task is `completed` or `failed`, then read `audio_url`, optional `video_url`, `title`, and `lyrics`.

## Model discovery

Music generation is asynchronous. Do not build against an upstream provider name or provider task ID; AI Sonar returns a normalized task identity and polling URL.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Create a music task

`action: "MUSIC"` creates songs or instrumentals depending on the prompt and model. `action: "LYRICS"` is for lyric-only flows when the selected model supports it. Use `continue_clip_id` and `continue_at` only for continuation workflows documented by the model contract.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Polling results

Always prefer the returned `poll_url`. The stable fallback is [Get task status](/api-reference/tasks/get-task-status). Completed music tasks can include `audio_url`, `video_url`, `title`, `lyrics`, and model-normalized metadata.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
