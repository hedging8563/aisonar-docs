## التدفق الموصى به

1. استدعِ [قائمة النماذج](/ar/api-reference/models/list-models) مع `recommended_for=music`.
2. استخدم [إنشاء موسيقى](/ar/api-reference/music/create-music) لإنشاء مهمة تحتوي على `prompt` وحقول اختيارية مثل `title` و `tags` و `action` و `mv` أو إعدادات المتابعة.
3. احفظ `id` / `task_id` و `poll_url` المُعادين.
4. استطلع حتى تصبح المهمة `completed` أو `failed`، ثم اقرأ `audio_url` و `video_url` الاختياري و `title` و `lyrics`.

## اكتشاف النماذج

إنشاء الموسيقى غير متزامن. لا تعتمد على اسم مزود upstream أو معرف مهمة المزود؛ يعيد AI Sonar هوية مهمة موحدة ورابط استطلاع.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## إنشاء مهمة موسيقى

`action: "MUSIC"` ينشئ أغاني أو مقاطع آلية حسب prompt والنموذج. `action: "LYRICS"` مخصص لتدفقات الكلمات فقط عندما يدعمه النموذج المختار. استخدم `continue_clip_id` و `continue_at` فقط في تدفقات المتابعة التي يوثقها عقد النموذج.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## استطلاع النتائج

فضّل دائماً `poll_url` المُعاد. نقطة fallback الثابتة هي [حالة المهمة](/ar/api-reference/tasks/get-task-status). قد تحتوي مهام الموسيقى المكتملة على `audio_url` و `video_url` و `title` و `lyrics` وبيانات وصفية موحدة.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
