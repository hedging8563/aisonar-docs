## التدفق الموصى به

1. استدعِ [قائمة النماذج](/ar/api-reference/models/list-models) مع `recommended_for=3d`.
2. استخدم [إنشاء 3D](/ar/api-reference/3d/create-3d) مع `prompt` نصي؛ أضف `image` أو `image_url` فقط للنماذج التي تدعم تحويل الصورة إلى 3D.
3. اختر الحقول الاختيارية مثل `format` و `quality` و `style` و `seed` حسب عقد النموذج المختار.
4. استطلع `poll_url` حتى يصبح الأصل جاهزاً، ثم نزّل `model_url` أو روابط الصيغ مثل `glb_url` / `fbx_url`.

## اكتشاف النماذج

إنشاء 3D يعتمد على المهام. اعتبر كتالوج النماذج مصدر الحقيقة بدلاً من افتراض مزود واحد أو نموذج ثابت.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## إنشاء مهمة 3D

طلبات النص إلى 3D تحتاج إلى `prompt`. وطلبات الصورة إلى 3D تضيف بيانات base64 في `image` أو `image_url`. استخدم `format` و `quality` فقط عندما يتيحهما عقد النموذج.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## استطلاع النتائج

استخدم `poll_url` المُعاد أولاً. نقطة fallback الثابتة هي [حالة المهمة](/ar/api-reference/tasks/get-task-status). ترجع مهام 3D المكتملة `model_url` وقد تتضمن `glb_url` أو `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
