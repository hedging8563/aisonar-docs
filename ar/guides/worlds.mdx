تنشئ عملية World عوالم Marble قابلة للاستكشاف بشكل غير متزامن. استخدم Worlds عندما يكون الناتج بيئة يمكن للمستخدم فتحها أو الرجوع إليها. استخدم توليد 3D عندما تحتاج إلى أصل نموذج مستقل.

## المدخلات وأصول الوسائط

يمكن أن تبدأ مهام World من نص أو صور أو صور متعددة أو فيديو حسب النموذج المحدد وشكل الطلب. للمطالبات الإعلامية القابلة لإعادة الاستخدام، جهّز media asset أولاً ثم أشر إليه في طلب World. تكون media assets وWorlds المكتملة متاحة لحسابك فقط.

## الإنشاء والمتابعة

تتضمن استجابات الإنشاء ID مهمة عامًا وغالبًا poll_url. احفظ هذا ID قبل عرض التقدم، ثم اتبع poll_url أو GET /v1/tasks/{id} حتى تصبح المهمة completed أو failed.

```bash
curl -X POST "https://api.aisonar.dev/v1/worlds/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "marble-1.1",
    "prompt": "A quiet coastal town at sunset with narrow alleys and warm lights"
  }'
```

## إدارة Worlds المكتملة

استخدم نقاط قائمة وتفاصيل Worlds للعوالم المكتملة المتاحة في حسابك. استخدم نقاط حالة المهمة للمهام المنتظرة أو الفاشلة. لا تحذف إلا عندما يزيل المستخدم World الناتج عمدًا من منتجك.

## أخطاء شائعة

- لا تفترض أن كل نموذج World يدعم كل نوع إدخال.
- أبقِ URLs الصور أو الفيديو الموقعة صالحة بما يكفي لمحاولات جلب الخدمة.
- احفظ ID المهمة الراجع قبل إعادة المحاولة بعد timeout.
- استخدم توليد 3D لملفات النماذج القابلة للتنزيل بدلاً من Worlds القابلة للاستكشاف.
