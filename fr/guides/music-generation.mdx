## Flux recommandé

1. Appelez [Lister les modèles](/fr/api-reference/models/list-models) avec `recommended_for=music`.
2. Utilisez [Créer de la musique](/fr/api-reference/music/create-music) pour créer une tâche avec `prompt` et, au besoin, `title`, `tags`, `action`, `mv` ou les paramètres de continuation.
3. Conservez le `id` / `task_id` et le `poll_url` renvoyés.
4. Interrogez jusqu’à `completed` ou `failed`, puis lisez `audio_url`, l’éventuel `video_url`, `title` et `lyrics`.

## Découvrir les modèles

La génération musicale est asynchrone. Ne dépendez pas d’un nom de fournisseur amont ou d’un ID de tâche fournisseur ; AI Sonar renvoie une identité de tâche normalisée et une URL de polling.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Créer une tâche musicale

`action: "MUSIC"` crée des chansons ou des instrumentaux selon le prompt et le modèle. `action: "LYRICS"` sert aux flux paroles seules lorsque le modèle le permet. Utilisez `continue_clip_id` et `continue_at` uniquement pour les continuations documentées par le contrat du modèle.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Polling des résultats

Privilégiez toujours le `poll_url` renvoyé. Le secours stable est [Statut de tâche](/fr/api-reference/tasks/get-task-status). Les tâches musicales terminées peuvent inclure `audio_url`, `video_url`, `title`, `lyrics` et des métadonnées normalisées.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
