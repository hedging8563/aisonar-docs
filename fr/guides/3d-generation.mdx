title: "Génération 3D"
description: "Créez des assets 3D depuis du texte ou des images avec les API 3D de AI Sonar et le polling asynchrone."
---
## Flux recommandé

1. Appelez [Lister les modèles](/fr/api-reference/models/list-models) avec `recommended_for=3d`.
2. Utilisez [Créer en 3D](/fr/api-reference/3d/create-3d) avec un `prompt` texte ; ajoutez `image` ou `image_url` uniquement pour les modèles image-vers-3D qui le prennent en charge.
3. Choisissez `format`, `quality`, `style` ou `seed` selon le contrat du modèle.
4. Interrogez `poll_url` jusqu’à ce que l’asset soit prêt, puis téléchargez `model_url` ou des URLs spécifiques comme `glb_url` / `fbx_url`.

## Découvrir les modèles

La génération 3D est basée sur des tâches. Utilisez le catalogue de modèles comme source de vérité au lieu de supposer un fournisseur ou un modèle fixe.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Créer une tâche 3D

Les requêtes texte-vers-3D nécessitent `prompt`. Les requêtes image-vers-3D ajoutent `image` en base64 ou `image_url`. Utilisez `format` et `quality` uniquement lorsque le contrat du modèle les expose.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Polling des résultats

Utilisez d’abord le `poll_url` renvoyé. Le secours stable est [Statut de tâche](/fr/api-reference/tasks/get-task-status). Les tâches 3D terminées renvoient `model_url` et peuvent inclure `glb_url` ou `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
