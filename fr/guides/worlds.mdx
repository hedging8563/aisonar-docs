La génération World crée de façon asynchrone des Marble Worlds explorables. Utilisez Worlds pour un environnement que les utilisateurs peuvent ouvrir ou revisiter. Utilisez la génération 3D quand vous avez besoin d’un modèle autonome.

## Entrées Et Assets Média

Les tâches World peuvent partir de texte, d’images, de plusieurs images ou d’une vidéo selon le modèle choisi et la forme de requête. Pour les prompts média réutilisables, préparez d’abord un media asset, puis référencez-le dans la requête World. Les media assets et Worlds terminés ne sont disponibles que pour votre compte.

## Créer Et Suivre

Les réponses de création incluent un ID de tâche public et généralement une poll_url. Stockez cet ID avant d’afficher la progression, puis suivez poll_url ou GET /v1/tasks/{id} jusqu’à completed ou failed.

```bash
curl -X POST "https://api.aisonar.dev/v1/worlds/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "marble-1.1",
    "prompt": "A quiet coastal town at sunset with narrow alleys and warm lights"
  }'
```

## Gérer Les Worlds Terminés

Utilisez les endpoints de liste et de détail Worlds pour les Worlds terminés disponibles dans votre compte. Utilisez les endpoints de statut de tâche pour les générations en attente ou échouées. Supprimez uniquement lorsque l’utilisateur retire volontairement le World généré de votre produit.

## Erreurs Fréquentes

- Ne supposez pas que chaque modèle World prend en charge tous les types d’entrée.
- Gardez les URLs signées d’image ou vidéo valides assez longtemps pour les tentatives de récupération du service.
- Stockez l’ID de tâche retourné avant de réessayer après un timeout.
- Utilisez la génération 3D pour les fichiers de modèle téléchargeables plutôt que pour des Worlds explorables.
