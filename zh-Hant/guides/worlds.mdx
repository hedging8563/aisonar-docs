World 生成會非同步建立可探索的 Marble World。當輸出是使用者可以開啟或再次訪問的環境時使用 Worlds；如果需要可下載的單獨模型資產，請使用 3D 生成。

## 輸入與媒體資產

World 任務可根據所選模型和請求形態從文字、圖片、多圖或影片開始。可重用的媒體提示應先準備媒體資產，再在 World 請求中引用。媒體資產和完成後的 World 只對你的帳戶可用。

## 建立與輪詢

建立響應包含公共任務 ID，通常也包含 poll_url。展示進度前先保存這個 ID，然後跟隨 poll_url 或 GET /v1/tasks/{id}，直到任務 completed 或 failed。

```bash
curl -X POST "https://api.aisonar.dev/v1/worlds/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "marble-1.1",
    "prompt": "A quiet coastal town at sunset with narrow alleys and warm lights"
  }'
```

## 管理已完成的 Worlds

Worlds 列表和詳情端點用於查看目前帳戶裡已完成的 World。待處理或失敗的生成任務請使用任務狀態端點。只有使用者明確刪除產品裡的生成 World 時才呼叫刪除。

## 常見問題

- 不要假設每個 World 模型都支援所有輸入類型。
- 簽名圖片或影片 URL 要保持足夠長的有效期，覆蓋服務端拉取重試。
- 逾時後重試前，先保存並查詢已返回的任務 ID。
- 需要可下載模型檔案時使用 3D 生成，而不是 Worlds。
