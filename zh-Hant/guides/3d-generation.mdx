## 建議流程

1. 调用 [模型列表](/zh-Hant/api-reference/models/list-models)，带上 `recommended_for=3d`。
2. 用 [创建 3D](/zh-Hant/api-reference/3d/create-3d) 发送文本 `prompt`；只有支持图生 3D 的模型才传 `image` 或 `image_url`。
3. 按模型契约选择 `format`、`quality`、`style`、`seed` 等可选字段。
4. 轮询 `poll_url` 到资产就绪，然后下载 `model_url` 或 `glb_url` / `fbx_url` 等格式化 URL。

## 模型發現

3D 生成是任务型 API。应以模型目录为真值，不要假设只有某一个供应商或固定模型。

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## 建立 3D 任務

文生 3D 请求需要 `prompt`。图生 3D 请求再增加 `image` base64 或 `image_url`。`format` 和 `quality` 只在模型契约列出时使用。

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## 輪詢結果

优先使用返回的 `poll_url`。稳定兜底入口是 [查询任务状态](/zh-Hant/api-reference/tasks/get-task-status)。完成后的 3D 任务返回 `model_url`，也可能包含 `glb_url` 或 `fbx_url` 等格式字段。

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
