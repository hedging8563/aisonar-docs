## Luồng đề xuất

1. Gọi [Danh sách mô hình](/vi/api-reference/models/list-models) với `recommended_for=3d`.
2. Dùng [Tạo 3D](/vi/api-reference/3d/create-3d) với `prompt` văn bản; chỉ thêm `image` hoặc `image_url` cho mô hình hỗ trợ ảnh sang 3D.
3. Chọn các trường tùy chọn như `format`, `quality`, `style`, `seed` theo hợp đồng mô hình.
4. Polling `poll_url` cho đến khi tài sản sẵn sàng, sau đó tải `model_url` hoặc URL theo định dạng như `glb_url` / `fbx_url`.

## Khám phá mô hình

Tạo 3D là API dạng tác vụ. Hãy xem catalog mô hình là nguồn đúng thay vì giả định một provider hay mô hình cố định.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Tạo tác vụ 3D

Request văn bản sang 3D cần `prompt`. Request ảnh sang 3D thêm dữ liệu base64 `image` hoặc `image_url`. Chỉ dùng `format` và `quality` khi hợp đồng mô hình có chúng.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Polling kết quả

Dùng `poll_url` được trả về trước. Endpoint dự phòng ổn định là [Trạng thái tác vụ](/vi/api-reference/tasks/get-task-status). Tác vụ 3D hoàn tất trả `model_url` và có thể có `glb_url` hoặc `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
