## Luồng đề xuất

1. Gọi [Danh sách mô hình](/vi/api-reference/models/list-models) với `recommended_for=music`.
2. Dùng [Tạo nhạc](/vi/api-reference/music/create-music) để tạo tác vụ với `prompt` và các trường tùy chọn như `title`, `tags`, `action`, `mv` hoặc thiết lập tiếp tục.
3. Lưu `id` / `task_id` và `poll_url` được trả về.
4. Polling cho đến khi tác vụ là `completed` hoặc `failed`, sau đó đọc `audio_url`, `video_url` tùy chọn, `title` và `lyrics`.

## Khám phá mô hình

Tạo nhạc là bất đồng bộ. Đừng phụ thuộc vào tên provider upstream hoặc task ID của provider; AI Sonar trả về định danh tác vụ đã chuẩn hóa và URL polling.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Tạo tác vụ nhạc

`action: "MUSIC"` tạo bài hát hoặc nhạc không lời tùy prompt và mô hình. `action: "LYRICS"` dành cho luồng chỉ tạo lời khi mô hình hỗ trợ. Chỉ dùng `continue_clip_id` và `continue_at` cho luồng tiếp tục được hợp đồng mô hình ghi rõ.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Polling kết quả

Luôn ưu tiên `poll_url` được trả về. Endpoint dự phòng ổn định là [Trạng thái tác vụ](/vi/api-reference/tasks/get-task-status). Tác vụ nhạc hoàn tất có thể gồm `audio_url`, `video_url`, `title`, `lyrics` và metadata đã chuẩn hóa.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
