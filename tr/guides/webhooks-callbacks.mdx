## Genel bakış

Bu kılavuz, uygulamanızı provider implementation detaylarına bağlamadan production’da Webhooks ve callback politikası kullanmayı açıklar.

AI Sonar public API’lerini, model metadata’sını ve dokümante edilmiş response field’larını sözleşme olarak kullanın.

## Güncel davranış

- Release öncesi bu değerleri kontrol edin: `task_id`, `poll_url`, public task durumu.
- Provider’a özel status URL’lerine, routing identifier’lara veya private execution metadata’ya bağlı kalmayın.
- Retry işlemlerini idempotent yapın ve çift billable work oluşturmayın.

## Production kontrol listesi

- Loglar support için yararlı olmalı, ancak secret, private file, prompt veya kullanıcı medyası saklamamalıdır.
- Support ticket; sanitize edilmiş request shape, endpoint, model, timestamp ve public identifier içermelidir.
- Production trafiğini taşımadan önce flow’u staging’de doğrulayın.
