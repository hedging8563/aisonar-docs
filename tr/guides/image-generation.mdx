## Önerilen akış

1. [Modelleri listele](/tr/api-reference/models/list-models) çağrısını `recommended_for=image` ile yapın.
2. Seçilen model için [Modeli getir](/tr/api-reference/models/get-model) sayfasını açın ve public contract alanını izleyin.
3. Metinden görsel ve desteklenen referans görsel akışları için [Görsel oluştur](/tr/api-reference/images/create-image), düzenleme akışları için [Görsel düzenle](/tr/api-reference/images/edit-image) kullanın.
4. Yanıtta `poll_url` varsa görev son duruma gelene kadar bu URL’i yoklayın.

## Modelleri keşfet

Görsel modellerinin request şekli aynı değildir. Bazıları sonucu inline döndürür, bazıları asenkron görev oluşturur. `size`, `quality`, `aspect_ratio`, `resolution`, `async` alanlarının kabul edilip edilmediğini seçilen model sözleşmesi belirler.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Metinden görsel

GPT Image ailesi modellerinde model sözleşmesi sunuyorsa `size` kullanın. Google veya Nano Banana görsel modellerinde `aspect_ratio` ve `resolution` yalnızca sözleşmede listelenmişse kullanılmalıdır.

```bash
curl https://api.aisonar.dev/v1/images/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-2",
    "prompt": "A clean product photo of a ceramic coffee cup on a walnut desk",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## Referans görseller ve düzenleme

`/v1/images/generations` üzerinde referans görselle üretim için model destekliyorsa `operation: "image-to-image"` ile `image_url`, `image_urls`, `reference_image_urls` veya multipart `image` dosyası gönderin. Multipart görselleri, görsel URL’lerini veya `images[].image_url` / `images[].file_id` alanlarını kabul eden düzenleme modelleri için `/v1/images/edits` kullanın.

Top-level `images[]` veya top-level `file_id` alanlarını `/v1/images/generations` endpointine göndermeyin; bu şekiller onları açıkça belgeleyen edit akışlarına aittir.

```bash
curl https://api.aisonar.dev/v1/images/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "nano-banana-pro",
    "operation": "image-to-image",
    "prompt": "Keep the same product, place it in a bright studio scene",
    "image_urls": ["https://example.com/reference.png"],
    "aspect_ratio": "1:1"
  }'
```

## Sonuçları yokla

Senkron görsel yanıtı `data[].url` veya `data[].b64_json` döndürür. Görev yanıtı `status`, `task_id` ve genellikle `poll_url` döndürür. `poll_url` önceliklidir; sabit fallback [Görev durumu](/tr/api-reference/tasks/get-task-status) endpointidir. Asenkron görsel görevleri URL döndürür, inline base64’i yalnızca desteklenen senkron akışlarda isteyin.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
