## Alur yang disarankan

1. Panggil [Daftar model](/id/api-reference/models/list-models) dengan `recommended_for=music`.
2. Gunakan [Buat musik](/id/api-reference/music/create-music) untuk membuat tugas dengan `prompt` dan field opsional seperti `title`, `tags`, `action`, `mv`, atau pengaturan lanjutan.
3. Simpan `id` / `task_id` dan `poll_url` yang dikembalikan.
4. Polling sampai tugas `completed` atau `failed`, lalu baca `audio_url`, `video_url` opsional, `title`, dan `lyrics`.

## Temukan model

Generasi musik bersifat asinkron. Jangan bergantung pada nama provider upstream atau ID tugas provider; AI Sonar mengembalikan identitas tugas yang dinormalisasi dan URL polling.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Buat tugas musik

`action: "MUSIC"` membuat lagu atau instrumental tergantung prompt dan model. `action: "LYRICS"` untuk alur lirik saja jika model mendukungnya. Gunakan `continue_clip_id` dan `continue_at` hanya untuk alur lanjutan yang didokumentasikan kontrak model.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Polling hasil

Selalu prioritaskan `poll_url` yang dikembalikan. Fallback stabilnya adalah [Status tugas](/id/api-reference/tasks/get-task-status). Tugas musik selesai dapat berisi `audio_url`, `video_url`, `title`, `lyrics`, dan metadata ternormalisasi.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
