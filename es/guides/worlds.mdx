title: "Worlds"
description: "Genera Worlds explorables, sube prompts multimedia, consulta tareas y administra Worlds completados."
---

La generación World crea Marble Worlds explorables de forma asíncrona. Usa Worlds cuando el resultado sea un entorno que los usuarios puedan abrir o revisar. Usa generación 3D cuando necesites un modelo independiente.

## Entradas Y Media Assets

Las tareas World pueden empezar desde texto, imágenes, varias imágenes o video según el modelo seleccionado y la forma de la solicitud. Para prompts multimedia reutilizables, prepara primero un media asset y luego referéncialo en la solicitud World. Los media assets y Worlds completados solo están disponibles para tu cuenta.

## Crear Y Consultar

Las respuestas de creación incluyen un ID público de tarea y normalmente una poll_url. Guarda ese ID antes de mostrar progreso y luego sigue poll_url o GET /v1/tasks/{id} hasta que la tarea sea completed o failed.

```bash
curl -X POST "https://api.aisonar.dev/v1/worlds/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "marble-1.1",
    "prompt": "A quiet coastal town at sunset with narrow alleys and warm lights"
  }'
```

## Administrar Worlds Completados

Usa los endpoints de lista y detalle de Worlds para Worlds completados disponibles en tu cuenta. Usa endpoints de estado de tarea para generaciones pendientes o fallidas. Elimina solo cuando el usuario quite intencionalmente el World generado de tu producto.

## Errores Comunes

- No asumas que todos los modelos World soportan todos los tipos de entrada.
- Mantén las URLs firmadas de imagen o video válidas el tiempo suficiente para reintentos de descarga del servicio.
- Guarda el ID de tarea devuelto antes de reintentar tras un timeout.
- Usa generación 3D para archivos de modelo descargables, no para Worlds explorables.
