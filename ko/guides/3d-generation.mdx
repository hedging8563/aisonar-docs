## 권장 흐름

1. [모델 목록](/ko/api-reference/models/list-models)을 `recommended_for=3d`와 함께 호출합니다.
2. 텍스트 `prompt`로 [3D 생성](/ko/api-reference/3d/create-3d)을 사용합니다. `image` 또는 `image_url`은 이미지-3D 모델이 지원할 때만 추가합니다.
3. `format`, `quality`, `style`, `seed` 같은 선택 필드는 선택한 모델 계약에 맞춰 사용합니다.
4. 자산이 준비될 때까지 `poll_url`을 폴링한 다음 `model_url` 또는 `glb_url` / `fbx_url` 같은 형식별 URL을 다운로드합니다.

## 모델 찾기

3D 생성은 작업 기반입니다. 하나의 제공자나 고정 모델을 가정하지 말고 모델 카탈로그를 기준으로 삼으세요.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## 3D 작업 생성

텍스트-3D 요청에는 `prompt`가 필요합니다. 이미지-3D 요청은 base64 `image` 또는 `image_url`을 추가합니다. `format`과 `quality`는 모델 계약에 있을 때만 사용합니다.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## 결과 폴링

먼저 반환된 `poll_url`을 사용하세요. 안정적인 대체 엔드포인트는 [작업 상태 조회](/ko/api-reference/tasks/get-task-status)입니다. 완료된 3D 작업은 `model_url`을 반환하며 `glb_url` 또는 `fbx_url`을 포함할 수 있습니다.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
