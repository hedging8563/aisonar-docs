## 개요

이 가이드는 애플리케이션을 provider 구현 세부 정보에 묶지 않고 모델 수명주기 및 사용 중단을 production에서 사용하는 방법을 설명합니다.

AI Sonar public API, model metadata, 문서화된 response field를 계약으로 사용하세요.

## 현재 동작

- 릴리스 전에 다음 값을 확인하세요: `/v1/models`, `/v1/pricing`, model ID.
- provider별 status URL, routing identifier, private execution metadata에 의존하지 마세요.
- retry는 idempotent하게 만들고 중복 과금 work를 만들지 마세요.

## 프로덕션 체크리스트

- log는 support에 유용하게 남기되 secret, private file, prompt, 사용자 media는 저장하지 마세요.
- support ticket에는 sanitizing된 request shape, endpoint, model, timestamp, public identifier를 포함하세요.
- production traffic을 전환하기 전에 staging에서 flow를 검증하세요.
