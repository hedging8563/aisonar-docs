A geração World cria Marble Worlds exploráveis de forma assíncrona. Use Worlds quando a saída for um ambiente que usuários podem abrir ou revisitar. Use geração 3D quando precisar de um asset de modelo independente.

## Entradas E Media Assets

Tarefas World podem começar com texto, imagens, várias imagens ou vídeo, dependendo do modelo selecionado e do formato da requisição. Para prompts de mídia reutilizáveis, prepare primeiro um media asset e depois referencie-o na requisição World. Media assets e Worlds concluídos ficam disponíveis apenas para sua conta.

## Criar E Acompanhar

Respostas de criação incluem um ID público de tarefa e geralmente uma poll_url. Salve esse ID antes de mostrar progresso e siga poll_url ou GET /v1/tasks/{id} até a tarefa ficar completed ou failed.

```bash
curl -X POST "https://api.aisonar.dev/v1/worlds/generations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "marble-1.1",
    "prompt": "A quiet coastal town at sunset with narrow alleys and warm lights"
  }'
```

## Gerenciar Worlds Concluídos

Use os endpoints de lista e detalhe de Worlds para Worlds concluídos disponíveis na sua conta. Use endpoints de status de tarefa para gerações pendentes ou com falha. Exclua apenas quando o usuário remover intencionalmente o World gerado do seu produto.

## Erros Comuns

- Não presuma que todo modelo World suporta todos os tipos de entrada.
- Mantenha URLs assinadas de imagem ou vídeo válidas por tempo suficiente para tentativas de busca do serviço.
- Salve o ID de tarefa retornado antes de tentar novamente após timeout.
- Use geração 3D para arquivos de modelo baixáveis, não para Worlds exploráveis.
