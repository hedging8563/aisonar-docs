---
## Fluxo recomendado

1. Chame [Listar modelos](/pt/api-reference/models/list-models) com `recommended_for=3d`.
2. Use [Criar 3D](/pt/api-reference/3d/create-3d) com um `prompt` de texto; adicione `image` ou `image_url` apenas para modelos que aceitam imagem para 3D.
3. Escolha campos opcionais como `format`, `quality`, `style` ou `seed` de acordo com o contrato do modelo selecionado.
4. Consulte `poll_url` até o ativo ficar pronto, depois baixe `model_url` ou URLs específicas como `glb_url` / `fbx_url`.

## Descobrir modelos

A geração 3D é baseada em tarefas. Use o catálogo de modelos como fonte de verdade em vez de assumir um provedor ou modelo fixo.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Criar uma tarefa 3D

Requisições texto para 3D exigem `prompt`. Requisições imagem para 3D adicionam `image` em base64 ou `image_url`. Use `format` e `quality` somente quando o contrato do modelo expuser esses campos.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Consultar resultados

Use primeiro o `poll_url` retornado. O fallback estável é [Status da tarefa](/pt/api-reference/tasks/get-task-status). Tarefas 3D concluídas retornam `model_url` e podem incluir `glb_url` ou `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
