## Fluxo recomendado

1. Chame [Listar modelos](/pt/api-reference/models/list-models) com `recommended_for=music`.
2. Use [Criar música](/pt/api-reference/music/create-music) para criar uma tarefa com `prompt` e campos opcionais como `title`, `tags`, `action`, `mv` ou configurações de continuação.
3. Guarde o `id` / `task_id` e o `poll_url` retornados.
4. Consulte até a tarefa ficar `completed` ou `failed`, depois leia `audio_url`, `video_url` opcional, `title` e `lyrics`.

## Descobrir modelos

A geração de música é assíncrona. Não dependa do nome do provedor upstream nem do ID de tarefa do provedor; a AI Sonar retorna uma identidade de tarefa normalizada e uma URL de polling.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Criar uma tarefa de música

`action: "MUSIC"` cria músicas ou instrumentais conforme o prompt e o modelo. `action: "LYRICS"` é para fluxos somente de letra quando o modelo selecionado aceita. Use `continue_clip_id` e `continue_at` apenas em fluxos de continuação documentados pelo contrato do modelo.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Consultar resultados

Prefira sempre o `poll_url` retornado. O fallback estável é [Status da tarefa](/pt/api-reference/tasks/get-task-status). Tarefas de música concluídas podem incluir `audio_url`, `video_url`, `title`, `lyrics` e metadados normalizados pelo modelo.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
